# 作品文件上传目录

## 目录结构（规范化）

```
uploads/
  └── works/                           # 作品文件根目录
      └── {年份}/                       # 按年份分类（如：2026）
          └── {赛事名称}/               # 按赛事分类（如：计算机设计大赛）
              └── {赛道类型}/           # 按赛道类型分类（如：CODE、PPT、VIDEO）
                  └── {作品编号}/       # 每个作品的专属文件夹（如：WORK-2026-001）
                      ├── {规范文件名} # 上传的文件（规范化命名）
                      └── ...
```

## 文件命名规范

### 文件名格式
```
{作品编号}_{团队编号}_v{版本号}_{文件类型}_{时间戳}.{扩展名}
```

### 示例
```
WORK-2026-001_TEAM-A1B2C3D_v1_SOURCE_20260411120530.zip
WORK-2026-002_TEAM-B2C3D4E5_v2_DOCUMENT_20260411121045.pdf
WORK-2026-003_TEAM-C3D4E5F6_v1_DEMO_20260411121520.mp4
```

### 组成部分说明

1. **作品编号** - 格式：`WORK-{年份}-{序号}`（如：WORK-2026-001）
   - 可快速识别作品
   - 年份和序号便于排序

2. **团队编号** - 格式：`TEAM-{UUID前8位}`（如：TEAM-A1B2C3D4）
   - 标识提交作品的团队
   - 便于追溯团队信息

3. **版本号** - 格式：`v{数字}`（如：v1、v2）
   - 标识作品的版本
   - 支持版本迭代管理

4. **文件类型** - 枚举值：
   - `SOURCE` - 源代码文件
   - `DOCUMENT` - 文档文件
   - `DEMO` - 演示视频
   - `OTHER` - 其他文件

5. **时间戳** - 格式：`yyyyMMddHHmmss`（如：20260411120530）
   - 精确到秒的上传时间
   - 避免文件名冲突
   - 便于按时间排序

6. **扩展名** - 原始文件扩展名（如：.zip、.pdf、.mp4）

## 文件存储路径规范

### 路径格式
```
uploads/works/{年份}/{赛事名称}/{赛道类型}/{作品编号}/{文件名}
```

### 实际路径示例
```
uploads/works/2026/计算机设计大赛/CODE/WORK-2026-001/WORK-2026-001_TEAM-A1B2C3D_v1_SOURCE_20260411120530.zip
uploads/works/2026/计算机设计大赛/PPT/WORK-2026-002/WORK-2026-002_TEAM-B2C3D4E5_v2_DOCUMENT_20260411121045.pdf
uploads/works/2026/计算机设计大赛/VIDEO/WORK-2026-003/WORK-2026-003_TEAM-C3D4E5F6_v1_DEMO_20260411121520.mp4
```

### 分层说明

1. **年份层** - 按年份分类，便于归档和清理历史数据
2. **赛事层** - 按赛事分类，便于区分不同赛事的作品
3. **赛道层** - 按赛道类型分类，便于按作品类型管理
4. **作品层** - 每个作品独立文件夹，便于整体打包和下载

## 数据库记录

数据库中保存两条文件名记录：

1. **原始文件名** (`file_name`) - 用于前端显示，方便用户识别
   - 示例：`我的项目源码.zip`

2. **存储路径** (`file_path`) - 实际存储的规范化文件路径
   - 示例：`uploads/works/2026/计算机设计大赛/CODE/WORK-2026-001/WORK-2026-001_TEAM-A1B2C3D_v1_SOURCE_20260411120530.zip`

## 文件分类

作品附件分为以下类型：

| 类型 | 说明 | 典型文件格式 |
|------|------|--------------|
| SOURCE | 源代码文件 | .zip, .rar, .java, .py, .js |
| DOCUMENT | 文档文件 | .pdf, .doc, .docx, .ppt |
| DEMO | 演示视频 | .mp4, .avi, .mov, .wmv |
| OTHER | 其他文件 | 其他所有格式 |

## 文件大小限制

- **单个文件最大**：300MB
- **总请求最大**：300MB

## 权限控制

只有作品的团队成员可以：
- 上传作品附件
- 查看附件列表
- 删除附件

## 文件管理优势

### 规范化命名优势

1. **可读性强** - 文件名包含完整信息，一目了然
2. **易于追溯** - 作品编号、团队编号、版本号清晰标识
3. **便于排序** - 时间戳和版本号便于按时间/版本管理
4. **避免冲突** - 时间戳精确到秒，避免重复
5. **规范统一** - 所有文件遵循统一命名规则

### 分层存储优势

1. **分类清晰** - 按年份、赛事、赛道分层，结构清晰
2. **易于管理** - 可以批量管理某一年的所有作品
3. **便于归档** - 历史作品按年份归档，不影响当前赛事
4. **快速定位** - 通过路径快速找到特定作品
5. **批量操作** - 可以批量下载某赛事某赛道所有作品

## 备份建议

定期备份 `uploads/works` 目录，建议：

1. **按年份备份** - 每年结束后备份当年的所有作品
2. **增量备份** - 每周增量备份最近的作品
3. **云存储备份** - 使用云存储服务双重备份

## 清理规则

当作品被删除时，相应的文件夹也会被删除（通过删除附件记录触发）。

建议定期清理：
- 超过5年的历史作品（根据归档策略）
- 未提交的草稿作品（超过1年）
