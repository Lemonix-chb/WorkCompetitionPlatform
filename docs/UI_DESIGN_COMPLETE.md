# 前端完整界面设计规划（基于Apple Design System）

## 设计系统核心原则

### 配色方案
- **背景交替**: 黑色(#000000) ↔ 浅灰(#f5f5f7) - 电影节奏
- **单一交互色**: Apple蓝(#0071e3) - 仅用于按钮和链接
- **文字颜色**: 白色(深色背景) / 近黑(#1d1d1f)(浅色背景)

### 字体层级
1. **Hero标题**: 56px, weight 600, line-height 1.07, letter-spacing -0.28px
2. **Section标题**: 40px, weight 600, line-height 1.10
3. **Tile标题**: 28px, weight 400, line-height 1.14
4. **正文**: 17px, weight 400, line-height 1.47, letter-spacing -0.374px
5. **Caption**: 14px, weight 400, line-height 1.29

### 布局原则
- **内容宽度**: 980px（居中）
- **间距**: 8px基础单位（8, 17, 24, 40, 64）
- **圆角**: 按钮8px, 卡片8px, 胶囊按钮980px
- **阴影**: rgba(0,0,0,0.22) 3px 5px 30px - 柔和摄影阴影

## 响应式设计策略

### 断点定义
- **Mobile**: < 640px (单列布局)
- **Tablet**: 640px - 1024px (双列布局)
- **Desktop**: > 1024px (三列布局)

### 字体缩放
| 断点 | Hero | Section | Tile | Body |
|------|------|---------|------|------|
| Desktop | 56px | 40px | 28px | 17px |
| Tablet | 40px | 28px | 21px | 17px |
| Mobile | 28px | 21px | 17px | 17px |

### 间距调整
- **Desktop**: 64px section padding
- **Tablet**: 40px section padding
- **Mobile**: 24px section padding

## 数据交互流程

### 获取数据示例
```javascript
// 获取赛事列表
fetch('/api/competitions')
  .then(res => res.json())
  .then(data => {
    competitions.value = data.data.records
  })

// 获取赛道详情
fetch(`/api/competitions/${id}/tracks`)
  .then(res => res.json())
  .then(data => {
    tracks.value = data.data
  })
```

### 用户操作流程
1. 访问首页 → 查看赛事列表
2. 点击"查看赛道" → 查看赛道详情
3. 选择赛道 → 跳转登录
4. 登录成功 → 跳转学生主页
5. 创建团队 → 邀请成员
6. 提交作品 → 上传文件
7. 查看评审 → 查看成绩
